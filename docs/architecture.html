03 / Hybrid Architecture SYSTEM DESIGN

Noetica leverages a high-performance Hybrid Architecture, utilizing Python for IO-bound orchestration and Zig for CPU-bound mathematical operations.

The Separation of Concerns

Python is excellent at making HTTP requests, interacting with the Google Sheets API, querying SQL databases, and interfacing with LLMs (Gemini). However, Python is notoriously slow at large-scale graph computation and vector mathematics.

Zig, on the other hand, is a systems programming language with manual memory management and bare-metal performance, making it the perfect candidate to execute our PageRank and TF-IDF matrix multiplications over thousands of discovery nodes.

The Orchestration Flow

1. Python Ingestion (`src/main.py`): Python triggers the V2 fetchers to gather raw JSON data from arXiv, Google Patents, and GitHub.

2. IPC Payload Transfer: Python dumps the aggregated corpus into `zig_engine/input.json`.

3. Zig Execution (`zig_engine`): Python spawns a subprocess calling the compiled Zig binary. Zig parses the JSON, constructs the Knowledge Graph in memory, calculates network centrality, and outputs `zig_engine/output.json`.

4. Python Synthesis: Python reads the finalized scores, injects them into the SQLite database, and prompts Gemini 1.5 Pro to synthesize the top results.
